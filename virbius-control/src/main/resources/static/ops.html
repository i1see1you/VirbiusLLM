累计定义（tb_cumulative）

定义滑动窗口与默认计数维度（CounterStore）；保存后刷新网关产物。须在「规则」中创建 runtime=cumulative，引用 cumulative_name 并配置 condition（阈值/compare）及 reason/risk/intent。Global 与 Route 级限流请使用不同 cumulative_name。按 Header/Query 累计：选 dimension var + 逻辑变量（须先在「上下文映射」配置，如 app_id ← X-App-Id）。

编辑

Route：配置了 uris 时仅按 URI匹配（来自 APISIX 当前 route，非客户端 Header）。Service 至少填一项 upstream / consumer / api_key_group。

Groovy 脚本实现 decide(ctx)，返回 true（命中）/ false（未命中）。 可用 ctx.listMatch(name)、ctx.listMatch(name, value)、ctx.getCumulative(name).count、ctx.var('logical')。 命中后 intent / risk / reason / enforce 取自规则行。

Lua 脚本实现 decide(ctx)，返回 boolean。可用全局 listMatch(name, value)、getCumulative(name).count、ctx.var('logical')。
