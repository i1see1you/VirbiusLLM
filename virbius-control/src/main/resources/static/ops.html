命名名单

名单仅定义 list_name + 维度 + 纯值条目；拦截/放行由脚本规则 listMatch(...) + 规则行 intent_action / risk_score 决定。按 Header/Query 匹配：维度选 var:逻辑变量（如 var:app_id），条目填纯值（如 evil），脚本写 listMatch('名单', ctx.var('app_id'))。

名单维度值创建过期备注

累计定义（tb_cumulative）

定义滑动窗口与默认计数维度（CounterStore）；保存后刷新网关产物。须在「规则」中创建 runtime=cumulative，引用 cumulative_name 并配置 condition（阈值/compare）及 reason/risk/intent。Global 与 Route 级限流请使用不同 cumulative_name。按 Header/Query 累计：选 dimension var + 逻辑变量（须先在「上下文映射」配置，如 app_id ← X-App-Id）。

名称维度窗口状态

编辑

var 维度从「上下文映射」的 vars 取值；未映射时该请求跳过 ingest/判定。

上下文映射（context_bindings）

逻辑名 → HTTP 来源；策略与名单只引用逻辑名。wire 留空时默认与逻辑名相同（如逻辑名 debug_flag → Query 参数 debug_flag）。

逻辑名来源 fromwire name / field

场景注册（scene_registry）

每个 scene_id 归属唯一 app_id；同一 app 可有多个 scene。运行时由 (app_id, uri, match) 解析 scene（见「请求映射」中的 app_id）。uris 须已在「网关路由」覆盖（支持 /path/*）。添加 / 删除 / 改 fail 开关会写入 DB；点同步到网关推产物。

scene_idapp_iddefaulturisprioritymatch query

网关路由（gateway.routes）

定义哪些 URI 进入网关 evaluate（精确 path 或末尾 * 前缀，如 /v1/chat/*）；scene 由「场景注册」解析。保存后需「同步到网关」推产物。

urimethods
